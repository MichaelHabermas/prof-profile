# Ideas

## Rule of Cool — Remaining Ideas

### #2 · View Source Easter Egg

A subtle `< />` link in the nav that intercepts the instinct every engineer has to check source. Clicking it opens an animated code panel showcasing a curated snippet of your most impressive production code — the Pixi.js canvas renderer, the Firebase dual-layer sync, the TF-IDF retriever. Engineers who read other engineers' portfolios always check the source; owning that behavior and turning it into a deliberate showcase is something no other portfolio does.

### #3 · Terminal Mode

A keyboard shortcut (`~` or `` ctrl+` ``) that opens a fake terminal overlay. Visitors type commands like `whoami`, `ls projects/`, `cat resume.txt`, `git log --oneline`. Returns real data from your portfolio. Engineers who discover it will screenshot it and share it. It's also a direct demonstration of your systems-layer comfort.

### #4 · Project Deep-Dive Modals

Clicking a project card expands it into a full case study overlay — architecture decisions, a key code snippet, a "what I'd do differently" section, lessons learned. Turns the "Selected Work" grid from a glance into a conversation. Especially powerful for Aurora Focus and CollabBoard where the technical depth is real and worth showing.

### #5 · Live GitHub Activity Feed

A small section or sidebar widget showing your real recent commit activity pulled from the GitHub API. Not just commit count — actual message titles, repos, timestamps. Proves you're actively building, not just maintaining a portfolio. For a hiring manager checking you out at 11pm, seeing "committed 3 hours ago" is worth more than any testimonial.

### #6 · Dark Mode Toggle

The iridescent palette would be *stunning* on a dark background — the same lavender/rose/sky gradients against near-black would look far more dramatic than on the current light gray. One toggle in the nav, preference stored in `localStorage`. Also just good craft signal.

### #7 · Availability / Status Indicator

A tasteful, subtle status badge — "Finishing Gauntlet AI · Available Aug 2026" — somewhere near the hero or footer CTA. Recruiters who land on your site need to know if you're even worth pursuing right now. Removing that ambiguity increases response rates. Can be a one-liner that's easy to update.

### #8 · "How I Build" Interactive Flow

A section showing your actual AI-augmented development workflow as an interactive diagram — Cursor → LangChain → Ollama → Pinecone → deploy. Not a list of tools, but a *process*. Hovering each node shows a sentence about how you use it in practice. Positions you as someone with a mature, opinionated approach to AI-assisted engineering, not just someone who listed the buzzwords.
